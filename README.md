Web-Library App

Web-Library App — веб-приложение для автоматизации работы библиотечного фонда образовательного учреждения.

Приложение позволяет просматривать каталог книг, выполнять поиск и фильтрацию, добавлять и редактировать книги, регистрировать читателей, оформлять выдачу и возврат литературы, а также управлять справочниками авторов, жанров и кабинетов.

## Основные возможности

* просмотр каталога книг;
* поиск книг по названию;
* фильтрация по году, жанру и наличию PDF;
* авторизация библиотекаря;
* добавление новых книг;
* редактирование информации о книгах;
* загрузка PDF-файлов книг через Cloudinary;
* регистрация читателей;
* выдача книг читателям;
* возврат книг;
* просмотр активных, просроченных и возвращённых выдач;
* управление авторами, жанрами и кабинетами.

## Требования для запуска

Перед запуском проекта должны быть установлены:

* Node.js;
* npm;

Проверить установку Node.js и npm можно командами:

```bash
node -v
npm -v
```

## Установка проекта

Склонируйте репозиторий:

```bash
git clone https://github.com/USERNAME/REPOSITORY.git
```

Перейдите в папку проекта (лучше в VScode):

```bash
cd REPOSITORY
```

Установите зависимости с помощью терминала:

```bash
npm install
```

После выполнения `npm install` автоматически установятся зависимости из `package.json`, включая Next.js, React, Supabase SDK и Cloudinary.

## Переменные окружения

В корне проекта необходимо указать данные ключей supabase и cloudinary файл:

```bash
.env.local
```

Добавьте в него переменные окружения (txt файл с ключами, паролями, и ссылкой на github):

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=

CLOUDINARY_CLOUD_NAME=
CLOUDINARY_API_KEY=
CLOUDINARY_API_SECRET=
```

## Запуск проекта

Для запуска проекта в режиме разработки выполните:

```bash
npm run dev
```

После запуска приложение будет доступно по адресу:

```bash
http://localhost:3000
```

## Сборка проекта

Для создания production-сборки выполните:

```bash
npm run build
```

Для запуска production-версии:

```bash
npm start
```

## Работа с базой данных

В проекте используется Supabase на базе PostgreSQL.

В базе данных используются таблицы для хранения:

* книг;
* авторов;
* жанров;
* кабинетов;
* экземпляров книг;
* читателей;
* выдач книг;
* пользователей системы.

## Работа с PDF-файлами

PDF-файлы книг загружаются через Cloudinary.

Cloudinary используется для хранения электронных версий книг. При добавлении или редактировании книги пользователь может загрузить PDF-файл, после чего ссылка на файл сохраняется в базе данных Supabase.

Рекомендуется ограничивать размер PDF-файлов, например до 10 MB.
